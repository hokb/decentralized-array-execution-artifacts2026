This experiment compares the execution speed of a simple, numerical expression when executed by various technologies. 
It runs the expression repeatedly and measures the time required for 1000 completed invocations. This process is 
repeated until the program ran for at least 10 seconds. All measured times are written into a file respectively. 
The measured timings are than collected by the last measurement (using the Virtual Processor of ILNumerics Accelerator 
on .NET) and integrated / joined into a plot. This plot should reproduce the experiment result from the paper. 

This experiment is designed to run on a Windows 10 (or later) desktop PC. It requires a multicore CPU with 4 or more cores. No GPU or similar is used. 

All technologies to compare with must be installed: 
* Intel Fortran Compiler and dependencies to compile and run the FORTRAN sources. (Alternatively, precompiled binaries are provided)
* Python3 to compile and run the numpy sources. Numba to JIT compile the NumPy sources. 
* .NET to compile and run the ILNumerics sources. 

In the following we describe the minimal steps required to compile all binaries / executables used in the experiment. 
We start by installing the software required for compiling the FORTRAN sources using Intel Fortran Compiler. 

( Note, in case that this FORTRAN compilation step must / can be skipped, precompiled x64 executables for Windows are provided, too. In this case  below steps 1,2, and 3 are skipped and you can proceed directly with step 4) for installing the FORTRAN runtimes. )

1) Download and install the standalone version of Intel Fortran Compiler 2025.1.0 
=================================================================================
As of writing this instruction the latest published version of the Fortran Compiler has been: 2025.1.0. 
In case a newer version is available, use the newer version instead. 

Note, the Intel compiler integrates with Visual Studio, if found at install time. 
However, Visual Studio is not required to compile this experiment! During the install integration into
Visual Studio can be deactivated. This will not influence the outcome of this experiment. 

* Visit the download site: 
  
https://www.intel.com/content/www/us/en/developer/tools/oneapi/fortran-compiler-download.html

* Ensure to have selected [Windows] as operating system and [Offline Installer] as installer type. 

* Click on: "Continue as a Guest (download starts immediately)" OR
  Enter your personal information (Email, Country) in the form and click on [Submit & Begin Download]

* After download (ca. 500 MB) completed, start the installer. Install the recommended items and use the standard installation configuration. This will take up to 10 minutes. 


2) Download and install Visual Studio Build Tools for Visual Studio 2022
=========================================================================
The build tools are required by Intel Fortran Compiler. They are commonly installed with Visual Studio. We install them directly, to prevent us from having to use a whole Visual Studio Installation. 

* The direct link to download the build tools: 

https://aka.ms/vs/17/release/vs_BuildTools.exe

ONLY, if above link is not working (anymore) start here: https://visualstudio.microsoft.com/de/downloads/
find "Buildtools für Visual Studio 2022" in the list. Select [Download] to start downloading. 

* Start the downloaded installer. It provides a GUI offering to select individual packages for install. 

* In the installer GUI select the following two (2) packages: 

+ Desktop development with C++
+ .NET desktop build tools

* Click on "Install" to start the installation. This may require up to 15 min.

3) Compile the FORTRAN sources of the experiment
================================================

* Run the "Intel oneAPI command prompt for Intel 64 for Visual Studio 2022". It can be accessed by pressing the [Win]-key 
and start typing "Intel oneAPI command". From the suggested list select the command prompt to start its setup script. 

* The command prompt configures environment variables needed to compile the sources. Ensure the start up display contains the following confirmation:
":: oneAPI environment initialized ::"

* navigate into the experiment directory (where the experiment files have been extracted to). 

* Navigate into the 1_FORTRAN sub-directory. 

 > cd 1_Fortran

* Execute the Compile.bat script in the oneAPI command prompt: 

 > Compile.bat

 This should compile the FORTRAN file 'Part1.f90' from the 1_FORTRAN directory and create two executable files in the same directory, overwriting existing files with the same name: 
+ Part1_F90_optim.exe and 
+ Part1_F90_nooptim.exe 
These binaries are later picked-up by the experiment for running the measurements of both FORTRAN targets. 

4) Install the Fortran runtime distributable package
====================================================

* Visit the Intel download site: 

https://www.intel.com/content/www/us/en/developer/articles/tool/compilers-redistributable-libraries-by-version.html

* Find the download link for the latest "Intel® Fortran Compiler Runtime for Windows* (IFX/IFORT)" package. Use the link to download and install the latest runtime binaries. 

* Find the download link for the latest "Intel® oneAPI DPC++/C++ Compiler Runtime for Windows*" package. Use the link to download and install the latest runtime binaries.

5) Install the Python3 framework
================================

* Download the latest Python 3 package for Windows 
https://www.python.org/downloads/

Alternatively, download the specific version 3.13.2 used for creating the plots in the paper: 
https://www.python.org/ftp/python/3.13.2/python-3.13.2-amd64.exe

* Install the Python package with default settings (includes pip)

* In a user command prompt (cmd.exe) run the following command: 

> pip install numpy numba

6) Install .NET 
=================

* Download the .NET SDK version 8.0 from: 
https://dotnet.microsoft.com/en-us/download/dotnet/8.0

* Install the SDK using default settings.
